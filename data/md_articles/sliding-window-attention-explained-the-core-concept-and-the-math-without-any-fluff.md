<p>This article is based on <strong>Longformer: The Long-Document Transformer</strong> (Beltagy, Peters &amp; Cohan, 2020, arXiv:2004.05150). The paper introduces a scalable attention mechanism for long documents; this article cuts the fluff and gets straight to the core concept, and the math, so you walk away with a complete, self-contained understanding of what Longformer actually does.</p><p>My advice: don’t read this article; just look at this image and you are good to go!</p><figure><img alt="" src="/images/blog/sliding-window-attention-explained-the-core-concept-and-the-math-without-any-fluff/hero.png" /><figcaption>Each grid reads as: row i = token i; cell (i,j) is filled if token i attends to token j. The four patterns shown are full n² attention, sliding window (w=3, attending 1 neighbor each side plus self), dilated sliding window (d=2, so attending self plus tokens 2 positions away on each side), and global + sliding window (token 0 is global: it attends to everything and everything attends to it, while all other tokens still use sliding window among themselves).</figcaption></figure><p>But if you want to learn some related concepts pertaining to sliding window attention then read the content below :)</p><h3>The problem: self-attention is quadratic</h3><p>Standard Transformer self-attention computes, for a sequence of length <strong>n</strong>, the following:</p><blockquote><em>STANDARD SELF ATTENTION</em></blockquote><blockquote>Attention(Q, K, V) = softmax( Q·Kᵀ / √d_k ) · V</blockquote><p>The bottleneck is <strong>Q·Kᵀ</strong>. Both Q and K have shape <strong>(n, d_k)</strong>, so their product is an <strong>(n × n)</strong> matrix. Time and memory both scale as <strong>O(n²)</strong>. For n = 512 this is manageable. For n = 4,096 or 16,384, it becomes either very slow or impossible on standard GPUs.</p><p>Longformer replaces this dense attention matrix with a sparse one, built from two components: a <strong>sliding window</strong> and a <strong>global attention</strong> layer.</p><h3>Component 1: sliding window attention</h3><p>Instead of letting every token attend to every other token, each token attends only to its neighbors within a fixed window of size <strong>w</strong>. Specifically, token <em>i</em> attends to tokens from index <strong>i — w/2</strong> to <strong>i + w/2</strong> (inclusive), that is <strong>w/2</strong> tokens to the left and <strong>w/2</strong> to the right.</p><blockquote><em>SLIDING WINDOW COMPLEXITY</em></blockquote><blockquote>O(n × w)</blockquote><p>Since <strong>w</strong> is a fixed constant (e.g. 512), this is linear in <strong>n</strong>. For a model with <strong>l</strong> layers, the <strong>receptive field</strong> at the top layer, meaning the span of input tokens each output token has indirectly seen, is:</p><blockquote><em>RECEPTIVE FIELD AFTER L LAYERS</em></blockquote><blockquote>l × w</blockquote><p>Each layer’s windowed attention feeds into the next, so representations from distant tokens gradually reach every position through stacking, exactly like how convolutional neural networks build global context through depth despite each kernel being local.</p><p>The paper also uses <strong>different window sizes per layer</strong>: smaller windows in the lower layers (which capture local syntax and token-level features) and larger windows in the higher layers (which build document-level representations). This balances compute with representation quality.</p><h3>Component 2: dilated sliding window</h3><p>To increase the receptive field without increasing compute, the window can be <strong>dilated</strong> by a gap factor <strong>d</strong>. Instead of attending to <em>contiguous</em> neighbors, token <em>i</em> attends to every <em>d</em>-th token within the window range, skipping the tokens in between.</p><blockquote><em>DILATED RECEPTIVE FIELD</em></blockquote><blockquote>l × d × w</blockquote><p>The number of tokens attended to per position remains <strong>w</strong> (fixed), so complexity stays <strong>O(n × w)</strong>. But because those w attended positions are spread out by factor d, a single layer already sees context that is d times wider.</p><p>In practice, the paper uses dilation only on <strong>2 out of 8 attention heads</strong> per layer. The remaining heads retain non-dilated (d = 1) windows. This lets some heads specialize in local context and others in longer-range context, which the ablation study confirms improves performance.</p><h3>Component 3: global attention</h3><p>[Bonus section; you can skip this]</p><p>Sliding window attention is good at building contextual representations locally. But some tasks need a token that has seen the <em>entire</em> sequence: classification needs a summary vector, and question answering needs the question tokens to be compared against every position in the document.</p><p>For these cases, Longformer paper adds <strong>global attention</strong> on a small set of pre-selected tokens. The rule is symmetric:</p><blockquote><em>GLOBAL ATTENTION RULE</em></blockquote><blockquote>A globally-attended token g attends to every token in the sequence (positions 1..n). Every token in the sequence also attends back to g.</blockquote><p>Let the number of globally-attended tokens be <strong>g</strong>. Since g is small and does not grow with n (it is determined by the task, e.g. 1 for a [CLS] token for a classification task), the added attention is <strong>O(n × g)</strong>. Combined with the sliding window:</p><blockquote><em>TOTAL COMPLEXITY</em></blockquote><blockquote>O(n × w) + O(n × g) = O(n) [since w and g are constants relative to n]</blockquote><h3>Two separate projection sets</h3><p>Because sliding window attention and global attention play structurally different roles, Longformer uses <strong>two separate sets of linear projection matrices</strong>:</p><blockquote><em>SLIDING WINDOW PROJECTIONS</em></blockquote><blockquote>Q_s, K_s, V_s → used for local windowed attention scores</blockquote><blockquote><em>GLOBAL ATTENTION PROJECTIONS</em></blockquote><blockquote>Q_g, K_g, V_g → used for global attention scores</blockquote><p>All six are initialized to the same values as the original pretrained model’s Q, K, V weights. The separation gives each attention type its own degrees of freedom during finetuning, which the WikiHop ablation study shows is critical: removing the separate projections for global attention drops accuracy by 1.6 points, and removing global attention entirely drops it by 8.3 points.</p><h3>The full picture in one place</h3><blockquote><em>STANDARD ATTENTION</em></blockquote><blockquote>Complexity: O(n²) | Every token attends to every token</blockquote><blockquote><em>LONGFORMER ATTENTION</em></blockquote><blockquote>For non-global token i:</blockquote><blockquote>- attends to tokens in [i — w/2, i + w/2] [sliding window]</blockquote><blockquote>- attends to all global tokens g₁, g₂, … [inbound global]</blockquote><blockquote>For global token g:</blockquote><blockquote>- attends to all n tokens in the sequence [outbound global]</blockquote><blockquote>Total complexity: O(n × w) + O(n × g) = O(n)</blockquote>
